Accessible names tests - Added 03/03/2021

The purpose these tests is to show the cascade order of how accessible names are applied in the accessibility tree for buttons and inputs. These examples should not be considered proper practice.

  • button where "aria-labelledby" should be accessible name
  • button where "aria-label" should be accessible name
  • button where content should be accessible name
  • button where "title" should be accessible name
  • button with no accessible name
  • input where "aria-labelledby" should be accessible name
  • input where "aria-label" should be accessible name
  • input where "label" should be accessible name
  • input where "title" should be accessible name
  • input with no accessible name
