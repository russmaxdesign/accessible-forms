<input> with visual "required" indicator

Return to main list

This is a test case designed to determine how a visual "required" indicator is announced by various screen readers.

Example

Code

<div>
  <label for="name">Full name (Required)</label>
  <input required type="text" id="name">
</div>

Assistive technologies

VoiceOver

  • Chrome: Full name. Required. Required. Invalid data. Edit text. [Required information announced twice]
  • Firefox: Full name. Required. Required. Invalid data. Edit text. [Required information announced twice]
  • Safari: Full name. Required. Required. Edit text. [Required information announced twice]

NVDA

  • Chrome: Full name. Required. Edit. Required. Invalid entry. Blank. [Required information announced twice]
  • Firefox: Full name. Required. Edit. Required. Invalid entry. Has autocomplete. Blank. [Required information announced twice]
  • Edge: Full name. Required. Edit. Required. Invalid entry. Blank. [Required information announced twice]

JAWS

  • Chrome: Full name. Left parent. Required. Right parent. Edit. Required. Invalid entry. Type in text. [Required information announced twice]
  • Firefox: Full name. Left parent. Required. Right parent. Edit. Required. Invalid entry. Type in text. [Required information announced twice]
  • Edge: Full name. Left parent. Required. Right parent. Edit. Required. Invalid entry. Type in text. [Required information announced twice]
